# Castor v1.2.0 实现归纳

> 目标：沉淀当前版本已实现能力，便于后续版本开发时快速建立上下文。  
> 说明：本文档的“v1.2.0”表示能力里程碑版本；仓库运行时版本号当前仍为 `1.0.0`（`package.json` / `src-tauri/tauri.conf.json`）。

## 1. 版本范围（v1.2.0）

在 `v1.0.0`（SSH + SFTP）基础上，本阶段重点完成了“服务器页的 systemd 部署管理子系统”，形成了从配置、部署、详情控制到日志诊断的闭环。

核心覆盖：

- 服务器页信息架构重构（左侧菜单 + 右侧内容）
- `systemd` 部署服务列表/新增/编辑/详情完整流程
- 部署配置持久化与远端应用（apply）
- 连接池复用与状态控制（启动/停止/重启）
- 日志读取（普通/实时/全屏）与轻量性能优化
- 从远端已有 systemd 服务导入配置
- 删除策略细化（远端卸载 + 仅删除本地配置）

## 2. 用户可见能力清单

## 2.1 服务器页面结构

- 左侧菜单：`服务器`、`设置`、`systemd部署`。
- 右侧内容区按菜单切换。
- systemd 相关操作集中在右侧区域，不再分散。

主要文件：
- `src/components/ServersView.tsx`
- `src/styles/app.css`

## 2.2 systemd 服务列表页

- 展示已保存的部署服务列表。
- 支持整卡点击进入详情（含键盘可达性）。
- 卡片操作保留：`详情`、`编辑`、`删除`。
- 列表页不主动查询运行状态，避免多服务场景下高频远端连接。

## 2.3 新增/编辑部署服务

- 表单字段覆盖：部署名称、目标服务器、服务名、scope、日志策略、目录、ExecStart/ExecStop、用户、环境变量、开机自启、sudo。
- `ExecStart`：按服务类型（Node/Python/Java/Go/.NET/Docker/自定义）展示示例，placeholder 随类型切换。
- 新增“仅保存”按钮，与“保存并部署”并存。
- 防重复提交：前端提交锁 + 后端唯一校验。
- 服务名唯一：前端即时提示 + 后端强校验（大小写不敏感，忽略 `.service` 后缀差异）。

## 2.4 详情页能力

- 展示配置与日志策略（含日志地址）。
- 操作按钮：`编辑`、`启动`、`停止`、`重启`、`刷新状态`、`删除`。
- 启动/停止互斥。
- 返回按钮不因“状态查询中”被禁用，仅在“删除进行中”禁用。

## 2.5 删除能力

- 删除前统一弹窗确认。
- 弹窗新增复选框：`仅删除本地配置（不删除远程服务）`，默认不勾选。
- 默认行为（不勾选）：删除本地配置并尝试卸载远端 unit。
- 勾选后：仅删除本地配置，远端服务保持不变。

## 2.6 日志能力（详情页）

- 按需读取：进入详情不自动拉日志；点击“查看日志”或“开启实时日志”后才请求。
- 实时日志：增量轮询 + 批量 flush（轻量性能版）。
- 全屏日志：支持全屏查看与 `Esc` 退出。
- 自动滚动：开启实时日志时自动滚到底部。
- 过滤能力（轻量版）：关键字输入 + 过滤按钮 + 清空 + 区分大小写（仅针对已加载日志）。
- 关键词高亮：过滤命中内容标红。
- 新增“清空当前日志”：只清空本地已加载日志与缓存，不影响远端日志文件。

## 2.7 日志输出策略

- 支持三种输出模式：
  - `journal`（默认）
  - `file`（指定路径）
  - `none`
- 默认日志路径随服务名联动：`/var/log/<服务名>.log`。
- 导入已有服务时可解析并回填日志模式与路径（避免被表单默认值覆盖）。

## 2.8 从已有 systemd 服务导入

- 新增“从已有服务导入”入口（创建页）。
- 支持读取目标服务器远端服务列表并导入 unit 关键配置到表单。
- 导入列表增强：
  - 仅展示常见“用户自建目录”中的服务（非全量系统服务）
  - 搜索过滤（服务名/状态）
  - 已存在于本地部署列表的服务标记“已添加”

## 3. 关键后端实现要点

## 3.1 持久化与并发安全

- 本地配置文件：`systemd_deploy_services.json`（应用配置目录）。
- upsert / delete 过程增加存储锁（`Mutex`）防止并发写入竞态。

核心文件：
- `src-tauri/src/deploy.rs`

## 3.2 连接池复用

- 基于 `(host, port, username, auth_fingerprint)` 的 SSH 会话连接池。
- 详情页频繁查询状态/日志/控制操作可复用同机连接，减少连接开销。

## 3.3 删除流程

- 默认远端卸载流程：`stop`、`disable`、删除 unit、`daemon-reload`、`reset-failed`。
- 通过 `remove_remote` 参数支持“仅删除本地配置”。

## 3.4 日志增量与去噪

- `journal` 模式：`journalctl --show-cursor --after-cursor`。
- `file` 模式：cursor 使用 `inode:offset`（兼容旧 `offset`）做增量读取。
- 修复点：`cursor == size` 时不重复回传历史尾部内容。
- 过滤输出噪声：`-- No entries --`、`-- Logs begin at ...`。
- ANSI 转义清理：剥离颜色控制码，避免前端显示乱码序列。

## 3.5 远端服务导入

- 新增命令：
  - `list_remote_systemd_services`
  - `get_remote_systemd_service_template`
- 导入解析字段：`Description`、`WorkingDirectory`、`ExecStart`、`ExecStop`、`User`、`Environment`、`StandardOutput/StandardError`。

## 6. 已知边界与后续建议

当前边界：

- 日志过滤仅作用于“已加载日志”；不是服务端全文检索。
- 实时日志为轻量方案（轮询 + 行数上限），尚未引入虚拟列表。
- “自建服务”识别基于常见目录规则（`/etc/systemd/system` / `$HOME/.config/systemd/user`），不等同于绝对准确的业务归属判断。

后续建议（v1.3.0 可选）：

1. 日志可观测性增强：匹配计数、上一条/下一条命中导航。  
2. 导入体验增强：导入前 diff 预览（覆盖哪些字段）。  
3. 删除体验增强：在“仅删本地”场景增加批量操作。  
4. 稳定性：补齐 deploy/log/import 流程自动化测试。  
5. 性能：实时日志在大数据量下切换到虚拟列表。
