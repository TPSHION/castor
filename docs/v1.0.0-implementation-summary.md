# Castor v1.0.0 实现归纳

> 目标：沉淀当前版本已实现能力，便于后续版本开发时快速建立上下文。  
> 说明：该文档记录的是 `v1.0.0` 阶段能力快照（历史版本归档）。

## 1. 版本范围（v1.0.0）

本版本已从“SSH MVP”扩展到“可用的桌面 SSH + SFTP 文件管理器”，核心覆盖：

- SSH 会话管理（远程 + 本地终端）
- 服务器配置管理（含持久化）
- 双栏 SFTP 文件浏览与文件操作
- 上传/下载任务并行、进度、取消、任务面板
- 文件冲突处理（自动重命名/覆盖/手动重命名）
- 拖拽传输（面板内拖拽 + 系统文件拖拽上传）
- 权限编辑（可视化开关 + 特殊权限说明）
- 标题栏融合与原生窗口拖拽能力

---

## 2. 用户可见功能清单

## 2.1 SSH 与会话

- 支持密码和私钥（PEM）认证连接 SSH。
- 支持“本地终端”会话（Local shell）。
- 支持多会话 Tab 展示、状态指示（connecting/connected/error/closed）。
- 支持会话重试、断开、关闭。
- 终端支持输入透传和 PTY 尺寸同步。

相关文件：
- `src/components/TerminalView.tsx`
- `src/components/WorkspaceView.tsx`
- `src-tauri/src/ssh/mod.rs`
- `src-tauri/src/commands.rs`

## 2.2 服务器配置管理

- 支持新增、编辑、删除、测试连接。
- 支持“快速连接”弹窗（包括快速打开本地终端）。
- 配置持久化到应用配置目录下 `connection_profiles.json`。
- 敏感字段（密码/私钥/口令）会持久化保存。

相关文件：
- `src/components/ServerEditorModal.tsx`
- `src/components/QuickConnectModal.tsx`
- `src/components/ServersView.tsx`
- `src-tauri/src/profiles.rs`

## 2.3 SFTP 双栏文件浏览（Local / Remote）

- 左右双栏布局，`Local` 与 `Remote (SFTP)` 并排展示。
- 本地和远程均支持：路径输入、打开、上级目录、刷新。
- 远程默认打开路径为 `/root`。
- 双击目录进入；远程双击文件可触发下载。
- 文件名/目录名过长时省略，避免挤压其他列。
- 表头与内容区域对齐；滚动区域限定在内容区（不覆盖表头）。

相关文件：
- `src/components/SftpView.tsx`
- `src/styles/app.css`

## 2.4 右键菜单与文件操作

- Local 与 Remote 均实现统一风格右键菜单。
- 均支持“空白区域右键菜单”。
- Local 菜单支持：打开目录、重命名、删除、刷新、新建文件夹、上传到目标目录。
- Remote 菜单支持：打开目录、重命名、删除、刷新、新建文件夹、下载到目标目录、编辑权限。

相关文件：
- `src/components/LocalContextMenu.tsx`
- `src/components/SftpContextMenu.tsx`
- `src/components/LocalActionDialog.tsx`
- `src/components/SftpActionDialog.tsx`

## 2.5 上传/下载任务系统

- 支持上传与下载进度展示（百分比、已传输/总量）。
- 支持预计剩余时间（ETA）和实时速度。
- 支持并行任务（无需等待前一个任务完成）。
- 支持“取消下载”和“取消上传”。
- 支持“最小展示时长”（600ms），避免进度条闪烁。
- 当任务较多时：主界面只展示一个进行中任务，其余任务通过“任务面板”查看。
- 任务面板区分进行中/已完成，支持“清空已完成”。

相关文件：
- `src/app/hooks/useTransferProgressManager.ts`
- `src/components/TransferTasksPanelModal.tsx`
- `src/components/SftpView.tsx`
- `src-tauri/src/sftp.rs`

## 2.6 传输冲突处理

- 上传时若远程已有同名项，弹窗提示冲突并给出建议。
- 提供三种策略：
  - 自动重命名上传（推荐）
  - 覆盖上传
  - 手动重命名上传
- 对文件/目录类型不一致的冲突提供风险提示。

相关文件：
- `src/components/LocalUploadConflictDialog.tsx`
- `src/App.tsx`
- `src-tauri/src/sftp.rs`（`SftpUploadConflictStrategy`）

## 2.7 拖拽上传/下载

- 支持 Local/Remote 面板间拖拽：
  - Local -> Remote 触发上传
  - Remote -> Local 触发下载
- 支持系统文件拖拽到 Remote 面板上传。
- 通过 Tauri 拖拽事件读取系统路径，上传触发后走 Rust 侧上传逻辑。
- 处理拖拽去重、队列与冲突弹窗联动。

相关文件：
- `src/components/SftpView.tsx`
- `src/app/hooks/useSystemDropUploadQueue.ts`
- `src-tauri/src/sftp.rs`

## 2.8 权限编辑（chmod）

- 远程“编辑权限”弹窗由数字输入改为开关式权限编辑：
  - 所有者/组/其他：读写执行开关
  - 特殊权限：SetUID / SetGID / Sticky
- 根据开关自动计算并展示八进制权限值。
- “?” 说明以悬浮提示方式提供解释。

相关文件：
- `src/components/SftpActionDialog.tsx`
- `src-tauri/src/sftp.rs`（`sftp_set_permissions`）

## 2.9 交互与样式体验优化

- 输入控件普遍禁用自动补全、自动纠错、自动大写、拼写检查。
- 拖拽过程中禁用页面文本选择（仅拖拽态）。
- 终端滚动条更细，且仅在滚动时显示。
- 顶部 Tab 区保持单行，超宽改为横向滚动（细滚动条策略）。
- 标题栏与系统红黄绿按钮同层融合（Tauri overlay），并处理全屏留白。
- 增加原生窗口拖动能力权限并在前端调用 `startDragging()` 兜底，降低“被 Tab 区挡住无法拖窗”问题。

相关文件：
- `src/styles/app.css`
- `src/components/TerminalView.tsx`
- `src/App.tsx`
- `src-tauri/tauri.conf.json`
- `src-tauri/capabilities/default.json`

## 3. 当前代码结构（重构后）

前端已从“单文件堆叠”拆分为模块化结构：

- 入口与总线：
  - `src/App.tsx`（页面编排与状态调度）
  - `src/types.ts`（跨层请求/响应类型）
- 视图组件：
  - `src/components/*`（SFTP、菜单、弹窗、任务面板、终端等）
- 业务辅助：
  - `src/app/helpers.ts`
  - `src/app/types.ts`
  - `src/app/hooks/*`（进度管理、系统拖拽队列、菜单关闭、滚动宽度）
- 样式：
  - `src/styles/app.css`

后端结构：

- `src-tauri/src/ssh/mod.rs`：SSH 会话生命周期与 IO
- `src-tauri/src/sftp.rs`：SFTP 文件操作、递归传输、进度事件、取消机制
- `src-tauri/src/localfs.rs`：本地文件系统浏览/重命名/删除/新建目录
- `src-tauri/src/profiles.rs`：连接配置持久化
- `src-tauri/src/commands.rs`：Tauri command 暴露层

---

## 4. 关键技术决策

- 使用 Tauri command + Rust 实现 SSH/SFTP 核心能力，前端仅做 UI 与状态管理。
- 传输进度通过事件 `sftp-transfer-progress` 回传前端。
- 上传/下载在 Rust 侧 `spawn_blocking`，支持并行任务。
- 标题栏采用 Tauri `titleBarStyle: Overlay` + `hiddenTitle`，并补充 `core:window:allow-start-dragging`。
- 右键菜单、弹窗、任务面板均组件化，便于迭代。

---

## 5. 数据与安全注意事项

- 连接配置保存在应用配置目录的 `connection_profiles.json`。
- 密码、私钥、私钥口令会被持久化（明文存储）。
- 生产化建议（下一版本可选）：
  - 引入系统钥匙串/安全存储（Keychain/Credential Manager）
  - 配置文件加密与主密码机制
  - 审计日志与敏感字段脱敏

## 7. 建议的 v1.1.0 优先方向

1. 安全：敏感信息改为安全存储（替代明文配置）。
2. 稳定性：补齐上传/下载/冲突/取消/拖拽相关测试。
3. 架构：继续拆分 `App.tsx`，引入更清晰的状态边界。
4. 体验：任务面板支持筛选、失败重试、批量操作。
5. 性能：前端分包与渲染优化，减少首次加载体积。
